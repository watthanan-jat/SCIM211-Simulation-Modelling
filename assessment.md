# Assessment

Assessment in SCIM211 Simulation Modelling is designed to evaluate both conceptual understanding and practical modelling ability.

## Assessment Components

Final assessment details may be confirmed by the instructor, but the course should normally include:

- Theory understanding: probability, simulation concepts, Monte Carlo methods, discrete-event simulation, Markov chains, and analysis of results.
- Lab work: progress on the Coffee Shop Simulation Project across the even-week labs.
- Project work: modelling decisions, code quality, experiment design, interpretation, and communication.
- Review or examination tasks: ability to solve problems and explain simulation methods clearly.

## Coffee Shop Simulation Project

The Coffee Shop Simulation Project is the main applied component of the course. Students will build the project gradually and should keep a clear record of:

- Model assumptions
- Input data or chosen distributions
- Random variable generation methods
- Simulation logic
- Performance measures
- Experiment results
- Interpretation and limitations

## Submission Expectations

Submitted work should be clear, organised, and reproducible. Code should run without unnecessary manual steps, and written explanations should make the modelling choices understandable to another student or instructor.

Students should submit their own work and be prepared to explain their code, calculations, and conclusions.

## Feedback

Feedback will focus on mathematical correctness, modelling judgment, code clarity, interpretation of results, and communication. Students are encouraged to use feedback from earlier labs to improve later stages of the project.
