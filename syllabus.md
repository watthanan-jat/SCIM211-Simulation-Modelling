# Syllabus

## Course Title

SCIM211 Simulation Modelling

## Target Students

This course is designed for second-year Industrial Mathematics and Data Science students.

## Course Description

Simulation modelling is used when real systems are too complex, expensive, risky, or time-consuming to study directly. In this course, students learn how to represent systems, generate random behaviour, run computational experiments, and interpret simulation output.

The course combines theory and applied work. Weekly theory sessions introduce key mathematical and modelling concepts, while even-week labs develop a semester-long Coffee Shop Simulation Project.

## Course Structure

- 15 teaching weeks
- 2 hours of theory every week
- 2 hours of lab every even week
- One continuing lab project: Coffee Shop Simulation Project

## Learning Outcomes

By the end of the course, students should be able to:

- Explain basic system, model, and simulation concepts.
- Use probability ideas to support simulation modelling.
- Generate random numbers and simulate common random variables.
- Design and run Monte Carlo simulation experiments.
- Build simple discrete-event simulations.
- Apply introductory Markov chain models.
- Analyse simulation input and output responsibly.
- Discuss uncertainty, assumptions, validation, and limitations in simulation results.
- Develop and communicate a small simulation project using clear code and clear reasoning.

## Main Topics

- Introduction to simulation modelling
- System concepts and model formulation
- Probability review for simulation
- Random number generation
- Simulation of random variables
- Monte Carlo simulation
- Discrete-event simulation
- Markov chains
- Input and output analysis
- Variance reduction
- Model validation
- Project reporting

## Lab Project

The lab component uses a Coffee Shop Simulation Project throughout the semester. Students will gradually model customer arrivals, service times, queues, staffing decisions, performance measures, and uncertainty in results.

Each lab builds on earlier work, so students should keep their code, notes, data, and results organised throughout the course.
